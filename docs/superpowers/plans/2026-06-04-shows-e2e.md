# Shows e2e Tests Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Покрыть полный жизненный цикл домена Shows e2e-тестами на Playwright против реального бэкенда ShowTail (:8000).

**Architecture:** Playwright с `testDir: ./e2e`. Отдельный `setup`-проект логинится через реальный UI и сохраняет `storageState`; основной `chromium`-проект переиспользует сессию. Один последовательный спек (`describe.serial`) прогоняет создание → навигацию из списка → edit → смену статуса → publish → результаты → документы и в `afterAll` удаляет выставку через `DELETE /api/shows/{id}`.

**Tech Stack:** Playwright (`@playwright/test`), TypeScript, существующий Next.js dev-сервер на :8082, бэкенд FastAPI на :8000.

**Предпосылки для прогона:** dev-сервер (`npm run dev`, :8082) и бэкенд (:8000, `/health/` = 200) должны быть запущены. Креды админа: `admin@admin.com` / `Password123!`.

**Справочные факты из кода (не менять, использовать в селекторах):**
- Логин: `/auth/jwt/sign-in`, поля `getByLabel('Email address')`, `getByLabel('Password')`, кнопка `getByRole('button', { name: 'Sign in' })`. После логина GuestGuard редиректит с `/auth` на дашборд.
- Токен в `localStorage['jwt_access_token']` (`src/auth/context/jwt/constant.ts`).
- Create/Edit форма (`src/sections/show/show-create-edit-form.tsx`): `getByLabel('Name')`, `getByLabel('Rank')` (MUI Select, первая реальная опция — не `—`), `getByLabel('Start date')`, `getByLabel('City')`, `getByLabel('Venue')`; submit `getByRole('button', { name: 'Create show' })` / `{ name: 'Save changes' }`. Тосты: `Create success!`, `Update success!`.
- Edit-вью (`src/sections/show/view/show-edit-view.tsx`): `getByLabel('Status')` (MUI select, опция `registration open`), кнопка `getByRole('button', { name: 'Publish' })`. Тост `Status updated!`. Sonner-тост: `[data-sonner-toast]`.
- Список (`show-list-view.tsx` + `show-table-toolbar.tsx`): фильтр города `getByPlaceholder('City')`; строка — ссылка с именем выставки (`getByRole('link', { name })`).
- Результаты: заголовок `Результаты — <name>`; Документы: заголовок `Документы — <name>`.
- axios baseURL `/api`; POST создаёт на `/api/shows`, publish на `/api/shows/{id}/publish`, delete на `/api/shows/{id}` (проксируется в бэкенд).

**Конфиг-факты:** vitest включает только `src/**/*.test.ts` (e2e не подхватит). tsconfig `include: ["**/*.ts"]` → `tsc --noEmit` проверит `e2e/**` и `playwright.config.ts` (после установки `@playwright/test` типы резолвятся — держать файлы типобезопасными). lint глобит только `src/**` (e2e не линтуется).

### Task 3: Auth setup-проект (health-check + UI-логин + storageState)

**Files:**
- Create: `e2e/auth.setup.ts`

- [ ] **Step 1: Создать e2e/auth.setup.ts**

```ts
import { test as setup, expect } from '@playwright/test';

// ----------------------------------------------------------------------

const AUTH_FILE = 'e2e/.auth/admin.json';
const BACKEND_HEALTH = process.env.E2E_BACKEND_HEALTH ?? 'http://localhost:8000/health/';

setup('authenticate as admin', async ({ page, request }) => {
  // Fail fast, with a clear message, if the backend is not up.
  const health = await request.get(BACKEND_HEALTH).catch(() => null);
  if (!health || !health.ok()) {
    throw new Error(
      `Backend not reachable at ${BACKEND_HEALTH}. Start ShowTail (:8000) before running e2e.`
    );
  }

  await page.goto('/auth/jwt/sign-in');
  await page.getByLabel('Email address').fill('admin@admin.com');
  await page.getByLabel('Password').fill('Password123!');
  await page.getByRole('button', { name: 'Sign in' }).click();

  // GuestGuard redirects authenticated users out of /auth into the dashboard.
  await page.waitForURL((url) => !url.pathname.startsWith('/auth'), { timeout: 30_000 });

  // Token is persisted to localStorage by the JWT auth context.
  await expect
    .poll(() => page.evaluate(() => localStorage.getItem('jwt_access_token')), { timeout: 15_000 })
    .not.toBeNull();

  await page.context().storageState({ path: AUTH_FILE });
});
```

- [ ] **Step 1: Создать e2e/shows.spec.ts с создателем и навигацией**

```ts
import { test, expect, request as apiRequest } from '@playwright/test';

// ----------------------------------------------------------------------

const BASE_URL = process.env.E2E_BASE_URL ?? 'http://localhost:8082';
const stamp = Date.now();
const showName = `E2E Show ${stamp}`;
const showCity = `E2E-City-${stamp}`;

let showId = '';
let accessToken: string | null = null;

test.describe.serial('Shows full lifecycle', () => {
  test('create a draft show', async ({ page }) => {
    await page.goto('/dashboard/shows/new');

    await page.getByLabel('Name').fill(showName);

    // Rank is a MUI Select; first option is the empty "—".
    await page.getByLabel('Rank').click();
    await page.getByRole('option').filter({ hasNotText: '—' }).first().click();

    await page.getByLabel('Start date').fill('2026-09-01');
    await page.getByLabel('City').fill(showCity);

    // Capture token (storageState already populated localStorage for this origin).
    accessToken = await page.evaluate(() => localStorage.getItem('jwt_access_token'));
    expect(accessToken).toBeTruthy();

    const createResp = page.waitForResponse(
      (r) => /\/api\/shows\/?$/.test(r.url()) && r.request().method() === 'POST'
    );
    await page.getByRole('button', { name: 'Create show' }).click();

    const resp = await createResp;
    expect(resp.ok()).toBeTruthy();
    const body = await resp.json();
    showId = body.id;
    expect(showId).toBeTruthy();

    await expect(page.getByText('Create success!')).toBeVisible();
    await page.waitForURL('**/dashboard/shows');
  });

  test('navigate from list to edit (404 regression)', async ({ page }) => {
    await page.goto('/dashboard/shows');
    await page.getByPlaceholder('City').fill(showCity);

    const rowLink = page.getByRole('link', { name: showName });
    await expect(rowLink).toBeVisible();
    await rowLink.click();

    await page.waitForURL(`**/dashboard/shows/${showId}/edit`);
    await expect(page.getByRole('button', { name: 'Save changes' })).toBeVisible();
  });
});
```

```ts
  });

  test.afterAll(async () => {
    if (!showId || !accessToken) return;
    // Hard-delete the show created by this run (DELETE /shows/{id} → 204).
    // Soft: failures here must not fail the suite.
    const ctx = await apiRequest.newContext();
    try {
      await ctx.delete(`${BASE_URL}/api/shows/${showId}`, {
        headers: { Authorization: `Bearer ${accessToken}` },
      });
    } catch {
      // ignore cleanup errors
    } finally {
      await ctx.dispose();
    }
  });
```

## Заметки по рискам (для исполнителя)
- **MUI Select**: если `getByLabel('Rank'/'Status').click()` не открывает список — кликнуть по `page.locator('[role="combobox"]')` рядом с лейблом или по самому полю; затем выбрать `getByRole('option', ...)`.
- **Гонка тост vs навигация** (edit/create): sonner-провайдер живёт на уровне приложения, тост переживает клиентскую навигацию — `expect(...).toBeVisible()` с дефолтным таймаутом ловит его.
- **Publish** может вернуть валидационную ошибку для свежего черновика — это ожидаемо, ассерт мягкий (тост любой природы).
- **Список не показывает строку**: фильтр по уникальному `showCity` сужает выдачу; если бэк-пагинация/сортировка прячет запись — проверить, что фильтр города реально ушёл в запрос (`/api/shows?city=...`).
- **Cleanup**: удаление идёт через прокси `/api/shows/{id}` с bearer-токеном; права админа зеркалят update — доступ есть.
