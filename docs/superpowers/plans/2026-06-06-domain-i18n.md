# Domain i18n (RU + EN) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Полностью интернационализировать наши доменные секции ShowTail (RU + EN), сделать русский языком по умолчанию, перевести наше боковое меню и breadcrumbs.

**Architecture:** Расширяем существующую i18next-инфраструктуру шаблона (lazy-load namespace'ов через `resourcesToBackend`, хук `useTranslate`). Заводим namespace на домен + общий `common`/`navbar`. Компоненты потребляют `useTranslate(['<domain>', 'common'])`. zod-схемы переводятся на фабрики `getXSchema(t)`. Каждый домен — изолированная единица, мёржится зелёной.

**Tech Stack:** Next.js 16 App Router, MUI 7, TypeScript, i18next + react-i18next + i18next-resources-to-backend, react-hook-form + zod, vitest.

**Спека:** `docs/superpowers/specs/2026-06-06-domain-i18n-design.md`

---

## Соглашения (общие для всех задач)

- **Гейты после КАЖДОЙ задачи** (CLAUDE.md): `npx tsc --noEmit` → 0; `npx eslint --fix <изменённые файлы>` затем `npm run lint` → 0; `npm test` → зелёно. Не коммитить, пока не зелёно.
- **JSON-ресурсы:** для каждого нового namespace создаём ДВА файла с идентичным набором ключей: `src/locales/langs/ru/<ns>.json` и `src/locales/langs/en/<ns>.json`. RU — целевой перевод, EN — исходные английские строки (которые сейчас в коде).
- **Потребление в клиентских компонентах:** `const { t } = useTranslate(['<domain>', 'common'])` из `src/locales`. Ключи домена — `t('list.title')`; общие — `t('common:actions.save')`.
- **Иконки/`paths`/значения backend-enum не меняем** — переводим только пользовательский текст.
- **Импорты:** после правок гнать `npx eslint --fix` (плагин `perfectionist` сортирует импорты).
- **Конвенции i18n (зафиксированы на эталоне dog, соблюдать во всех доменах):**
  - Лейбл чипа результата фильтра по поиску — переиспользуемый ключ `common:keyword` (`"Keyword"` / `"Ключевое слово"`), НЕ `common:actions.search` (там значение-плейсхолдер `"Search..."`).
  - **Без мёртвых ключей:** в `<ns>.json` добавляем только реально используемые ключи. Не плодим «на будущее».
  - Внутри `.map((t) => ...)` переименовывать переменную (например в `item`/`title`), чтобы не затенять функцию перевода `t`.

В компоненте `const schema = useMemo(() => getXSchema(t), [t]); resolver: zodResolver(schema)`. Обновить импорты типов в файлах-потребителях.

Починить unit-тесты.** Тесты схем (`__tests__/*-schema*.test.ts`) вызывают фабрику с identity-`t`: `const t = ((k: string) => k) as any; const schema = getXSchema(t);` — проверяют логику, не текст. Тесты utils (`*-utils.test.ts`) трогать только если они проверяли захардкоженные подписи; чистая логика остаётся.
