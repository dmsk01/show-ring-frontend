# Доработки бэкенда ShowTail для публичной витрины

Дата: 2026-06-03
Аудитория: команда бэкенда (репозиторий `E:\Coding\python-animal-platform`, FastAPI, `:8000`)
Связано: `docs/superpowers/specs/2026-06-03-public-showcase-design.md`

## Контекст

Фронт строит публичную витрину без логина: **питомники**, **объявления (животные)**, **выставки** — сетки карточек + детальные страницы. GET-эндпоинты уже отдают данные анонимно (200). Ниже — доработки, которых не хватает для полноценного UX. Витрина запускается и без них (с заглушками и клиентскими обходами), доработки повышают качество и снимают костыли.

Приоритеты: **P1** — заметно влияет на UX/ценность, **P2** — важно, **P3** — желательно.

---

## P1. Фото животных (собак)

**Проблема.** У `Dog` нет изображений (`DogResponse` не содержит фото). Карточки и деталь собаки вынуждены показывать заглушку. Это главное, что «удешевляет» витрину питомников.

**Запрос.**
1. Добавить в модель `Dog` главное фото и/или галерею:
   - `avatar_file_id: UUID | null` (главное фото, по аналогии с `Kennel.avatar_file_id`), и/или
   - `photo_file_ids: list[UUID]` (галерея).
2. Отразить поля в `DogResponse`, `DogCreate`, `DogUpdate`.
3. Эндпоинт привязки фото (один из вариантов):
   - переиспользовать общий `POST /files/upload` + установка `avatar_file_id` через `PUT /dogs/{id}`, либо
   - доменный `POST /dogs/{id}/images` (как уже есть `POST /classifieds/{id}/images`).

**Ожидаемый ответ (пример DogResponse):**
```json
{ "id": "…", "name": "…", "avatar_file_id": "…", "photo_file_ids": ["…"], "...": "…" }
```
Файлы отдаются публично через существующий `GET /files/{file_id}` — фронт сам строит URL (`fileUrl`).

**Что разблокирует:** реальные фото в tour-карточках и галерее деталей собаки вместо placeholder.

---

## P2. Публичный фильтр объявлений по статусу

**Проблема.** Витрина «Животные» должна показывать только активные объявления (`status='active'`), но `GET /classifieds` не принимает `status`. Сейчас фронт фильтрует на клиенте — это ломает пагинацию (страница может содержать неактивные).

**Запрос.** Добавить query-параметр `status` в `GET /classifieds` (и/или отдавать анонимно только `active` по умолчанию). Также подтвердить, что анонимный листинг **не** возвращает `moderation/closed/archived`.

**Что разблокирует:** корректная серверная пагинация публичного каталога объявлений.

---

## P2. Серверная сортировка списков

**Проблема.** Нет параметров сортировки у `GET /kennels`, `GET /dogs`, `GET /classifieds`, `GET /shows`. Фронт сортирует только загруженную страницу — при многостраничных выборках порядок некорректен.

**Запрос.** Добавить `sort_by` и `order` (`asc|desc`):
- kennels: `name`, `created_at`;
- dogs: `name`, `date_of_birth`, `created_at`;
- classifieds: `created_at`, `price`, `views_count`;
- shows: `date_start`, `created_at`.

**Что разблокирует:** корректную сортировку витрин на всех страницах.

---

## P2. Флаг верификации питомника в публичном ответе

**Проблема.** Метку «проверенный питомник» негде взять: `KennelResponse` не содержит флага (он есть только в `KennelModerationItem.is_verified`).

**Запрос.** Добавить `is_verified: bool` в `KennelResponse`.

**Что разблокирует:** значок доверия на карточке/детали питомника.

---

## P3. Связь «помёт → щенки»

**Проблема.** У `Dog` нет `litter_id`, а у `Litter` нет списка щенков. В карточке питомника помёт показывает только родителей, не самих щенков.

**Запрос.** Один из вариантов:
- `litter_id: UUID | null` в `Dog` (+ фильтр `GET /dogs?litter_id=`), либо
- `GET /litters/{id}/puppies` → список `DogResponse`.

**Что разблокирует:** показ щенков помёта в объявлении и связку «объявление о помёте ↔ конкретные собаки».

---

## P3. Развёрнутые ссылки на родителей в помёте

**Проблема.** `LitterResponse` отдаёт `father_id`/`mother_id`, и фронт делает доп. запрос `/dogs/{id}` ради имени родителя.

**Запрос.** Возвращать краткие объекты `father`/`mother`: `{ "id": "…", "name": "…", "avatar_file_id": "…" }`.

**Что разблокирует:** меньше запросов, имена родителей сразу в карточке помёта.

---

## P3. Счётчики у питомника

**Проблема.** Чтобы показать «N собак, M помётов» на карточке питомника в сетке, нужны доп. запросы.

**Запрос.** Добавить агрегаты в `KennelResponse`: `dogs_count: int`, `litters_count: int` (и при желании `active_classifieds_count`).

**Что разблокирует:** информативные карточки сетки без N+1 запросов.
