# Спека: адаптация страниц «О нас», «Контакты», «FAQ» под Show Ring

Дата: 2026-06-08

## Цель

Заменить демо-контент Minimal Kit на публичных маркетинговых страницах
`/about-us`, `/contact-us`, `/faqs` на контент, отражающий специфику платформы
**Show Ring** (выставки собак и кошек по правилам РКФ / FCI; роли заводчик /
организатор / участник; бесплатная регистрация). Заодно убрать нерелевантные
секции/блоки демо.

## Решения (согласованы с пользователем)

1. **Язык: i18n RU+EN** — как в доменах. Новые namespace'ы, RU — дефолт/фолбэк.
2. **Глубина: контент + чистка секций** — переписать тексты И удалить/заменить
   нерелевантные блоки. Картинки-обложки и демо-аватары остаются плейсхолдерами.
3. **Контакты: плейсхолдеры** — `info@showring.ru`, `support@showring.ru`,
   телефон-плейсхолдер, часы поддержки, соцсети-заглушки.
4. **Карта на «Контактах» заменяется** на блок контактной информации.

## i18n-структура

Конвенция доменов (`src/locales/langs/{en,ru}/<ns>.json`, авто-загрузка через
`resourcesToBackend`, наборы ключей en/ru идентичны, без мёртвых ключей):

- Новые namespace'ы: `about`, `contact`, `faqs` (en + ru).
- В `common.json` (en + ru) добавить блок `form`:
  `name`, `email`, `subject`, `message`, `submit` — переиспользуют `ContactForm`
  и `FaqsForm`.
- Потребление: `const { t } = useTranslate(['<ns>', 'common'])`. Общие подписи —
  через `common:form.*`.
- Массивы (FAQ-пункты, отзывы, категории, каналы связи) хранятся в JSON массивами
  и читаются через `t('key', { returnObjects: true })` с приведением типа.
- **Каждый** редактируемый section-файл, вызывающий `useTranslate`, получает
  `'use client';` первой строкой (рантайм-баг App Router; tsc/lint его не ловят).
- Не переводим имена API-полей, `paths.*`, иконки; только отображаемый текст.

## Затрагиваемые файлы

- Новые: `src/locales/langs/{en,ru}/{about,contact,faqs}.json`;
  `src/sections/contact/contact-info.tsx`.
- Изменяются: `common.json` (en+ru, +`form`); все перечисленные section-файлы;
  `src/sections/contact/view/contact-view.tsx`.
- Удаляется использование: `_testimonials`, `_mapContact`, `_faqs` (мок) на этих
  страницах. Сам `contact-map.tsx` можно оставить (демо-компонент), но не
  импортировать; либо удалить — решим на этапе плана (по умолчанию: не трогаем,
  просто перестаём импортировать).
