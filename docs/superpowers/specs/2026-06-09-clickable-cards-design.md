# Дизайн: расширение зоны клика доменных карточек

Дата: 2026-06-09
Ветка: feat/animals-filters

## Проблема

Доменные карточки сущностей (собаки, питомники, выставки, объявления, помёты)
имеют кликабельным только заголовок. Это плохой UX/UX-таргет: пользователю
нужно попасть точно в текст ссылки. Требование — сделать кликабельной всю
площадь карточки, при этом сохранив настоящую ссылку (`<a href>`) ради
открытия в новой вкладке, копирования ссылки, SEO и доступности.

## Решение: «растянутая ссылка» (stretched link)

Заголовок остаётся настоящей `<Link component={RouterLink} href>`. Его зона
клика растягивается на всю карточку через прозрачный псевдоэлемент:

```ts
'&::after': { content: '""', position: 'absolute', inset: 0, borderRadius: 'inherit' }
```

Карточка-контейнер получает `position: relative`, чтобы оверлей
позиционировался относительно неё. Вложенные интерактивные элементы (ссылки на
отца/мать в карточке помёта) поднимаются над оверлеем через
`position: relative` и остаются самостоятельно кликабельными.

Отвергнутая альтернатива — JS-навигация (`role="link"` + `onClick` +
`router.push`, текущий паттерн `KennelLitterCard`): не даёт настоящего `href`
(не работают средний клик / открыть в новой вкладке / копировать ссылку),
`role="link"` без `href` слабее по доступности. Принято решение
унифицировать проект на технику растянутой ссылки.

## Переиспользуемый помощник

Новый модуль `src/components/card-link/` (с `index.ts`-реэкспортом по
конвенции `src/components/*`), чтобы не дублировать логику оверлея в 5 файлах:

- **`CardLink`** — обёртка заголовка. Рендерит
  `<Link component={RouterLink} href sx={stretchSx, ...sx}>`. Пробрасывает
  `href`, `children`, `color`, `variant`, `sx`. Используется и внутри
  `ListItemText.primary` (Dog/Classified/Kennel), и как самостоятельный
  заголовок (Show/KennelLitter).
- **`cardActionableSx`** — sx для самого `Card`:
  - `position: 'relative'`
  - `cursor: 'pointer'`
  - `transition` на `box-shadow`
  - `'&:hover': { boxShadow: customShadows.z16 }` — ховер-аффорданс
    кликабельности (как в текущем `KennelLitterCard`).
- **`nestedActionableSx`** (или применение `{ position: 'relative' }` напрямую)
  — для подъёма вложенных ссылок над оверлеем.

`stretchSx` (псевдоэлемент `::after`) инкапсулирован внутри `CardLink`.

## Изменения по карточкам

| Файл | Изменение |
|------|-----------|
| `src/sections/dog/dog-card.tsx` | `Card` → `cardActionableSx`; заголовок → `CardLink`. Вложенных ссылок нет. |
| `src/sections/classified/classified-card.tsx` | то же; `Label`-метки некликабельны, конфликтов нет. |
| `src/sections/kennel/kennel-card.tsx` | то же. |
| `src/sections/show/show-card.tsx` | `Card` → `cardActionableSx` (уже имеет `p:3`); заголовок-`Link` → `CardLink`. |
| `src/sections/kennel/kennel-litter-card.tsx` | Убрать `role="link"`, `tabIndex`, `onClick`, `onKeyDown`, `useRouter`. Заголовок-помёт (`Typography subtitle1`) → `CardLink`. Ссылки отца/матери: убрать `onClick={stopPropagation}`, добавить `sx={{ position: 'relative' }}`. `Card` → `cardActionableSx`. |

Карточки собак переиспользуются для щенков помёта через `DogCardGrid`
(`litter-detail-view.tsx`), поэтому правка `DogCard` автоматически покрывает
сетку щенков.

## Вне объёма

Демо-карточки Minimal Kit (`product`, `job`, `tour`, `user`, `blog`, и т.д.) —
шаблонные демо, не доменные сущности проекта. Не трогаем.

## Доступность

- На карточке остаётся ровно одна доступная ссылка (заголовок) → один таб-стоп,
  понятный текст ссылки.
- Вложенные ссылки (отец/мать) — отдельные таб-стопы, сохраняют свои `href`.
- Оверлей прозрачен и не перехватывает выделение текста за пределами карточки.

## Критерии готовности

- `npx tsc --noEmit` → 0 ошибок.
- `npm run lint` → 0 ошибок (импорты отсортированы `perfectionist`).
- `npm test` → зелёно.
- Рантайм (если бэкенд `:8000/health/` отвечает 200): наведение показывает
  тень + `pointer`; клик по любой части карточки ведёт на детальную страницу;
  средний клик / Ctrl+клик открывает в новой вкладке; ссылки отца/матери в
  карточке помёта ведут на своих собак.
